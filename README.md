# Software Engineering Assignment

## Project: Hospital Management System (HMS)

---

## 1. Introduction
A **Hospital Management System (HMS)** is a software application designed to manage the day-to-day operations of a hospital. It handles functionalities such as patient registration, appointment scheduling, doctor management, billing, medical records, and report generation.

This system is **complex and critical**, as it deals with sensitive patient data and supports life-related services. Therefore, it requires a development approach that ensures reliability, safety, and continuous evaluation.

---

## 2. Selected SDLC Model
### **Spiral Model**

---

## 3. Justification (Why Spiral Model is Suitable)

1. **High Risk Management**  
   HMS involves sensitive data and critical operations. The Spiral model focuses on **risk identification and mitigation** at every stage.

2. **Handles Complexity Effectively**  
   The system includes multiple modules like billing, appointments, and records. Spiral allows **step-by-step development with continuous refinement**.

3. **Early Error Detection**  
   Each cycle includes testing and evaluation, which helps in identifying errors early and reducing system failure risks.

4. **User Feedback Integration**  
   Doctors, nurses, and administrative staff can provide feedback after each iteration, improving system usability and performance.

5. **Flexibility with Structured Approach**  
   Spiral provides flexibility like iterative models but also maintains a **structured and controlled process**, which is essential for healthcare systems.

---

## 4. Comparison with Other Models

### Waterfall Model (Not Suitable)
- Requires **fixed requirements** at the beginning  
- No flexibility once development starts  
- Testing happens late, increasing risk in critical systems  

### Agile Model (Less Suitable)
- Focuses on flexibility but **lacks strong risk analysis**  
- Frequent changes may affect system stability  
- Not ideal for systems requiring strict validation and control

## 6. Conclusion
The **Spiral Model** is the most suitable SDLC model for a Hospital Management System due to its strong emphasis on **risk management, iterative development, and continuous evaluation**. These features ensure the system remains reliable, secure, and efficient, which is essential for a critical healthcare environment.
