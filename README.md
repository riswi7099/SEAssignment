# Software Engineering Assignment  

## Project: Hospital Management System (HMS)

---

## 1. Introduction  
A **Hospital Management System (HMS)** is a comprehensive software suite designed to manage all aspects of a hospital's operations. This includes patient registration, electronic health records (EHR), billing, pharmacy inventory, laboratory results, and staff scheduling.  

Because this system handles life-critical data and sensitive personal information, any failure can have severe consequences. Therefore, thorough planning and risk assessment are essential.

---

## 2. Selected SDLC Model  
### Spiral Model  

---

## 3. Justification  

The Spiral Model is particularly suitable for a **complex and high-risk system like HMS** due to the following reasons:

- **Intense Risk Analysis**  
  In a hospital environment, risks such as data breaches and system failures must be minimized. The Spiral model includes risk assessment in every cycle, ensuring early detection and resolution.

- **Complexity Management**  
  HMS involves multiple departments like Radiology, ICU, and Billing. The model supports modular development, handling complexity step-by-step.

- **User Involvement and Feedback**  
  Doctors and administrative staff can provide feedback after each iteration, improving usability and system performance.

- **Flexibility and Customization**  
  Healthcare regulations (e.g., HIPAA, GDPR) may change. The Spiral model allows updates without disrupting the entire system.

- **High Reliability**  
  Continuous testing and evaluation in every phase ensure the system remains stable, secure, and reliable.

---

## 4. Comparison with Other Models  

### Waterfall Model (Not Suitable)  
- No flexibility after initial requirements  
- Late testing increases risk  
- Fixing errors at later stages is costly and time-consuming  

### Agile Model (Less Suitable)  
- Focuses on speed but lacks strong risk analysis  
- May not provide sufficient documentation for healthcare compliance  
- Not ideal for highly critical systems requiring strict validation  

---

---

## 6. Conclusion  

The **Spiral Model** is the most suitable SDLC model for a Hospital Management System because it emphasizes **risk management, iterative development, and continuous validation**.  

This ensures the system is **secure, reliable, and compliant**, which is essential in a healthcare environment where software directly impacts patient safety.
